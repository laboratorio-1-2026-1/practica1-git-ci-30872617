# Práctica de Evaluación: Gestión de Ramas y Flujo Colaborativo

### Propósito de la Práctica
[cite_start]El objetivo principal de este proyecto es validar la capacidad técnica en el uso de **Git** y **GitHub** para la gestión del ciclo de vida de un desarrollo de software[cite: 2]. 

Esta práctica se enfoca en:
* [cite_start]**Inicialización de repositorios** locales y vinculación con remotos[cite: 9, 12].
* [cite_start]**Gestión de ramas (branching)** para simular un entorno de desarrollo real mediante el uso de una rama `develop`[cite: 15, 16].
* [cite_start]**Integración de cambios (merge)** desde ramas de desarrollo hacia la rama principal (`main`) para la puesta en producción[cite: 22, 24].
* [cite_start]**Sincronización remota** constante para garantizar la integridad del código en la nube[cite: 13, 25].
