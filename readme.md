# Práctica de Evaluación: Gestión de Ramas y Flujo Colaborativo

### Propósito de la Práctica
El objetivo principal de este proyecto es validar la capacidad técnica en el uso de **Git** y **GitHub** para la gestión del ciclo de vida de un desarrollo de software.

Esta práctica se enfoca en:
**Inicialización de repositorios** locales y vinculación con remotos.
**Gestión de ramas (branching)** para simular un entorno de desarrollo real mediante el uso de una rama `develop`.
**Integración de cambios (merge)** desde ramas de desarrollo hacia la rama principal (`main`) para la puesta en producción.
**Sincronización remota** constante para garantizar la integridad del código en la nube.

### Estructura del Proyecto
`index.html`: Estructura básica con información personal y tablas de herramientas.
`README.md`: Documentación del propósito y flujo de trabajo.
